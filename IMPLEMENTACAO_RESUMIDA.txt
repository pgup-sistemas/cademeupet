PROBLEMA ORIGINAL
─────────────────────────────────────────────────────────────────────────────
Quando um usuário fazia um pagamento PIX:
  ❌ O pagamento era confirmado pelo EFI Bank
  ❌ A página NÃO atualiza para mostrar o sucesso
  ❌ Usuário fica preso na tela do PIX indefinidamente
  ❌ Mesmo com Ctrl+Shift+F5 não funciona

CAUSA RAIZ
─────────────────────────────────────────────────────────────────────────────
  A página verifica o status da doação UMA VEZ ao carregar.
  Depois nunca mais verifica, mesmo que o pagamento seja confirmado.

SOLUÇÃO IMPLEMENTADA
─────────────────────────────────────────────────────────────────────────────
  ✅ Polling automático que verifica status a cada 5 segundos
  ✅ JavaScript faz requisições AJAX silenciosas
  ✅ Quando status muda para "aprovada", página auto-recarrega
  ✅ Botão manual para forçar verificação a qualquer momento
  ✅ Timeout de 10 minutos se pagamento demorar
  ✅ Fallback ao webhook falhar

═══════════════════════════════════════════════════════════════════════════════

FLUXO DE FUNCIONAMENTO
─────────────────────────────────────────────────────────────────────────────

1. Usuário acessa /doacao-pix?id=32
                    ↓
2. Página carrega e verifica status UMA VEZ
                    ↓
3. Se não aprovado, inicia polling automático
                    ↓
4. A cada 5 segundos:
   - JavaScript faz POST para /api/status-doacao.php
   - Endpoint faz GET na API EFI
   - Retorna status atual da doação
                    ↓
5. Webhook EFI é chamado (em paralelo):
   - EFI envia notificação de pagamento
   - Webhook atualiza banco de dados
                    ↓
6. Polling detecta mudança:
   - Status = "aprovada"
   - Para de fazer requisições
   - Recarrega página (location.reload())
                    ↓
7. Página mostra:
   - "Pagamento confirmado!"
   - ✅ Usuário vê sucesso

═══════════════════════════════════════════════════════════════════════════════
 🛠️ TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

PROBLEMA: Polling não inicia
SOLUÇÃO:
  - Verifique se JavaScript está na página: F12 → Sources
  - Verifique se status inicial não é "aprovada"
  - Abra Console e procure por erros

PROBLEMA: Endpoint retorna 404
SOLUÇÃO:
  - Verifique se /api/status-doacao.php foi uploadado
  - Verifique permissões (chmod 644)
  - Teste com curl: curl -X POST https://seu-site/api/status-doacao.php

PROBLEMA: Status não atualiza mesmo com polling
SOLUÇÃO:
  - Webhook pode estar falhando
  - Verifique token EFI em config.php
  - Verifique logs: grep "efi-webhook" includes/petfinder_error_log
  - Teste sincronizarStatusDoacaoPix manualmente

PROBLEMA: Página fica carregando infinitamente
SOLUÇÃO:
  - Timeout foi atingido (10 minutos)
  - Webhook falhou e API EFI não respondeu
  - Verificar logs de erro no servidor
  - Testar conexão com API EFI
